# Vela conformance test vectors

Machine-readable input/output pairs for verifying alternative implementations of the Vela protocol. Each JSON file contains an array of test cases with `input` and `expected_output` fields.

## Running

```bash
vela conformance tests/conformance/
```

The runner reads every `.json` file in the given directory, executes each test case against Vela's actual implementation, and reports pass/fail.

## Format

Every test vector file follows this structure:

```json
{
  "suite": "name-of-suite",
  "description": "What this suite tests",
  "cases": [
    {
      "name": "human-readable test name",
      "input": { ... },
      "expected_output": { ... }
    }
  ]
}
```

## Determinism

All test vectors are deterministic. The conformance runner does not make network calls or use LLM inference. Time-dependent tests (recency scoring) use fixed reference years in their inputs.

## Generating vectors

The expected outputs were generated by running `cargo test gen_conformance_ids -- --ignored --nocapture` and extracting IDs from the actual Rust implementation. To regenerate after a protocol change, update the ignored test in `src/bundle.rs` and rerun.
