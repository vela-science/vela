# Vela proof trace format

`proof-trace.json` is the machine-readable audit record written by `vela proof`.
It records which frontier state, packet files, validation result, and caveats
produced a proof packet. It also records the canonical event log hash,
proposal-state hash, and replay status for the frontier snapshot used to build
the packet.

The trace covers source/evidence packet artifacts too:
`sources/source-registry.json`, `evidence/evidence-atoms.json`, and
`evidence/source-evidence-map.json`, plus condition artifacts
`conditions/condition-records.json` and `conditions/condition-matrix.json`.
Packet validation fails if those files are missing or if evidence atoms
reference missing source, finding, or condition IDs.

The trace is a reproducibility artifact. It is not a scientific conclusion.

## Location

```bash
vela proof frontier.json --out proof-packet
```

writes:

```text
proof-packet/proof-trace.json
```

`vela proof --json` also returns `trace_path`.

## Contract

The v0 trace is a single JSON object:

```json
{
  "trace_version": "0.2.0",
  "generated_at": "2026-04-22T00:00:00Z",
  "source": "frontier.json",
  "source_hash": "794b1c...",
  "snapshot_hash": "sha256:...",
  "event_log_hash": "sha256:...",
  "proposal_state_hash": "sha256:...",
  "replay_status": "ok",
  "schema_version": "0.2.0",
  "checked_artifacts": [
    "manifest.json",
    "overview.json",
    "signals.json",
    "review-queue.json",
    "quality-table.json",
    "state-transitions.json",
    "events/events.json",
    "events/replay-report.json",
    "proposals/proposals.json",
    "conditions/condition-records.json",
    "conditions/condition-matrix.json",
    "ro-crate-metadata.jsonld",
    "findings/full.json",
    "reviews/review-events.json",
    "reviews/confidence-updates.json"
  ],
  "caveats": [
    "Candidate contradictions, gaps, and bridges require human review.",
    "PubMed prior-art checks are rough signals, not proof of novelty.",
    "Retraction impact is simulated over declared dependency links."
  ],
  "status": "ok"
}
```

## Fields

- `trace_version`: trace contract version, currently `0.2.0`.
- `generated_at`: packet generation time.
- `source`: input frontier path.
- `source_hash`: SHA-256 hash of the frontier source. File inputs hash file bytes; directory inputs hash sorted relative paths and bytes.
- `snapshot_hash`: deterministic hash of the materialized frontier snapshot with non-state signing fields removed.
- `event_log_hash`: deterministic hash of the canonical event log.
- `proposal_state_hash`: deterministic hash of the persisted proposal list.
- `replay_status`: `ok` or `no_events` for valid packets; any conflict blocks packet validation.
- `schema_version`: frontier schema version, currently `0.2.0`.
- `checked_artifacts`: packet files required by the proof validator, including
  the canonical event log, replay report, combined state-transition projection,
  and review/confidence projection files.
- `caveats`: required conservative interpretation notes.
- `status`: `ok` when packet validation passed.
- `trace_path`: path to this trace file.

## Determinism

For the same frontier source and command arguments, stable fields remain stable.
The packet manifest may include generated timestamps, so `packet_manifest_hash`
is optional in v0. When present, packet validation verifies it against
`manifest.json`.

Release checks validate that the trace exists, parses as JSON, includes required
fields, and was generated by the current proof command.
