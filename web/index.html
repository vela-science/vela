Vela

A git-native protocol for replayable scientific frontier state.

Apache-2.0 · Rust · macOS · Linux

What it is

Vela turns a folder of papers into a bounded, reviewable state that a scientist or an agent can correct, replay, and sign. The release primitive is not the paper. It is the frontier: a state snapshot made of findings, evidence, conditions, and provenance, with a write path — proposal, accepted canonical event, reducer — that makes corrections part of the record.

The core write path is narrow. A finding bundle (assertion, evidence, conditions, entities, confidence, provenance, links) is the primary object. A proposal is the public write boundary. An accepted proposal becomes a canonical event, which replays deterministically into the frontier. A proof packet seals the current state so another party can re-verify it offline.

v0 proves state, not extraction. Lab runtime, federation network, autonomous agent loops, hubs, desktop apps — those stay roadmap. v0's claim is small and checkable: scientific work can become inspectable, correctable, replayable frontier state.

01 · install

One binary, one command.

$ git clone https://github.com/vela-science/vela && cd vela $ cargo build --release -p vela-protocol $ ./target/release/vela --help # or, from source tree: $ ./install.sh

Requires Rust 1.78+. No network calls required for local corpora. See the Quick start for LLM backend setup (optional).

02 · first frontier

Compile, review, replay, seal.

1$vela compile ./papers --output frontier.json
2$vela check frontier.json --strict --json
3$vela normalize frontier.json --out frontier.normalized.json
4$FINDING_ID=$(jq -r '.findings[0].id' frontier.json)
5$vela review frontier.normalized.json "$FINDING_ID" \
    --status contested --reason "Mouse-only evidence" \
    --reviewer reviewer:demo --apply
6$vela history frontier.normalized.json "$FINDING_ID"
7$vela proof frontier.normalized.json --out proof-packet
8$vela serve frontier.normalized.json

The proof moment: a reviewed correction becomes canonical history, the inherited frontier changes, the prior proof packet is marked stale, and a refreshed packet seals the corrected state. That path is the v0 claim. Run ./demo/v0-state-proof-demo.sh to watch it execute end to end.

03 · the bundle

Six nouns, no more.

Frontier
A bounded, reviewable frontier state over a scientific question.
Finding bundle
The primary state object — an assertion with evidence, conditions, entities, confidence, provenance, and links.
Evidence
The exact span, row, table, measurement, run, metric, or weak provenance unit bearing on a finding.
Condition
The boundary where a claim stops — species, assay, comparator, exposure, endpoint, translation scope.
Canonical event
The authoritative state-transition record. Every durable write lands here, via a proposal.
Proof packet
A portable review artifact — manifest, source and evidence tables, signals, event/replay data, proposals, proof trace.

04 · proof

A sealed, reviewable artifact.

Every proof packet is a folder of pinned JSON plus a replayable trace. Another party can verify it offline — no network, no trust in the sender, no hidden state. If a reviewed correction lands upstream, the old packet is marked stale and the refresh is mechanical.

Manifest
packet.lock.json · overview.json · scope.json
Sources
source-table.json (N entries, pinned hashes)
Evidence
evidence-matrix.json (per-finding provenance)
Signals
signals.json (gaps, bridges, tensions)
Events
canonical event log + replay trace
Quality
quality-table.json · frontier-quality.md
Trace
proof-trace.json (replay-to-frontier walk)
Reviewer human · ed25519 · time & identity
Agent optional · ed25519 · model & tool lineage
